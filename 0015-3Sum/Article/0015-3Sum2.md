The question comes from question No. 15 on LeetCode: The sum of three numbers.

### Title description

Given an array `nums` containing *n* integers, determine whether there are three elements *a, b, c in `nums` such that *a + b + c =* 0? Find all triplets that satisfy the condition and are not repeated.

**Note:** Answers cannot contain duplicate triples.

#### Example

```
Given the array nums = [-1, 0, 1, 2, -1, -4],

The set of triples that meets the requirements is:
[
  [-1, 0, 1],
  [-1, -1, 2]
]
```

### Question analysis

The easiest thing to think of is the triple cycle brute force search, the time complexity is `O(n^3)`. It is a bit high, please optimize it.

From the question, we learned that the main problem lies in `searching all situations that satisfy the conditions` and `avoiding duplicates`, then we can use `ascending array + double pointer` to effectively handle the problem and reduce the time complexity.

You may be wondering why you would choose to use this solution?

First of all, the time complexity of array sorting can reach `O(NlogN)`, which is acceptable to us. In addition, according to the characteristics of ordered arrays, duplicate items in the array will be close together, and duplicate items can be skipped without additional space storage. Since it is in ascending order, when the leftmost value is found to be greater than 0, it can be jumped out in time to end the operation.

Double pointers can be used for `dimension reduction`. By traversing the array, the current subscript value is taken as `fixed value`. The double pointers represent the `first and last values` of the sub-array behind the `fixed value`. The sum of the three values ​​is judged by continuously approaching the double pointers.

The specific algorithm flow is as follows:

1. Special judgment: For array length `n`, if the array is `null` or the array length is less than `3`, return `[ ]`;
2. Array sorting in ascending order;
3. Traverse the array:
   - If `num[i] > 0`: Because it is in ascending order, the result cannot be equal to 0, and the result is returned directly;
   - Let the left pointer `L = i + 1`, the right pointer `R = n - 1`, when `L < R`, execute the loop:
     - When `nums[i] + nums[L] + nums[R] == 0`, execute a loop to determine whether the left pointer and right pointer overlap with the next position, and `remove duplicate solutions`. And move `L,R` to the next position at the same time to find a new solution;
     - If `sum` is greater than `0`, it means that `nums[R]` is too large, and the `R pointer` is moved to the left
     - If `and` is less than `0`, it means `nums[L]` is too small, and the `L pointer` is moved right

### Reference code

```javascript
 // lang = JavaScript
var threeSum = function(nums) {
    let res = [];
    if (nums == null || nums.length < 3) {
        return res;
    }
    const len = nums.length;
    nums.sort((a, b) => a - b); // ascending order
    for (let i = 0; i < len - 2;) {
        const element = nums[i];
        if (element > 0) {
            // If the current number is greater than 0, the sum of the three numbers must be greater than 0, so the loop ends
            break;
        }  
        let L = i + 1,
            R = len - 1;
        while (L < R) {
            const sum = element + nums[L] + nums[R];
            if (sum == 0) {
                res.push([element, nums[L], nums[R]]);
                // Deduplication of left and right pointers & L+1 & R-1
                while (L < R && nums[L] == nums[++L]);
                while (L < R && nums[R] == nums[--R]);
            }else if (sum < 0) {
                while (L < R && nums[L] == nums[++L]);
            }else {
                while (L < R && nums[R] == nums[--R]);
            }
        }
        // Fixed value deduplication
        while (nums[i] == nums[++i]);
    }
    return res;
};
```

### Complexity analysis

- Time complexity: `O(n^2)`

  Array sorting `O(NlogN)`, array traversal `O(n)`, double pointer traversal `O(n)`, the overall complexity is `O(NlogN) + O(n) * O(n)`, `O(n^2)`

- Space complexity: `O(1)`
