### Title description

The [Hamming distance](https://baike.baidu.com/item/Hamming distance/475174?fr=aladdin) of two integers refers to the number of different corresponding bits in the binary digits of these two numbers.

Calculate the sum of Hamming distances between any two numbers in an array.

Example:

```
Input: 4, 14, 2

Output: 6

Explanation: In binary representation, 4 is represented as 0100, 14 is represented as 1110, and 2 is represented as 0010. (This expression is to reflect the relationship between the last four digits)
So the answer is:
HammingDistance(4, 14) + HammingDistance(4, 2) + HammingDistance(14, 2) = 2 + 2 + 2 = 6.
```

**Notice:**

1. The range of elements in the array is from `0` to `10^9`.
2. The length of the array does not exceed `10^4`.

### Question analysis

As an example, the time complexity of pairwise brute force calculation is O(n^2). There is definitely no problem in implementation, but when the amount of data is large, the performance is worrying.

We first write the array and the resulting digital binary number

```
 4    0 1 0 0
14    1 1 1 0
 2    0 0 1 0
HammingDistance(4, 14) = 1 0 1 0
HammingDistance(4, 2)  = 0 1 1 0
HammingDistance(14, 2) = 1 1 0 0
```

Combining the results, we can find a relationship by observing the binary system of these three numbers and the binary system of the operation result from left to right in columns:

Number of digits Count = 3

First column: 0 1 0 ==> 1 * (3 -1) = 2 = 1 0 1

> There is only one 1 in this column, which means that among the first digits of all numbers, the first digits of (Count - 1) numbers are different from **this number**, that is, when calculating the distance, the result is 1, that is, the number of 1's generated by this digit is 1 * (3 -1)

Second column: 1 1 0 ==> 2 * (3 -2) = 2 = 0 1 1

> There are 2 1s in this column, which means that among the second digits of all numbers, there are (Count - 2) numbers whose second digits are different from these **two numbers**, that is, the number of 1s generated by this digit is (Count - 2) + (Count - 2) = 2 * (3 - 2)

The third column is the same as the second column

Fourth column: 0 0 0 ==> 0 * (3 -0) = 0 = 0 0 0

> All the numbers in this column are the same, so 1 will not be generated when calculating the distance, and the result will be 0
>
> If it is 1 1 1, the same is true, 3 * (3 - 3), the result is still 0

Summary: The number of 1s generated by calculating the distance in each column = the number of 1s in this column * (number of numbers – the number of 1s in this column) = the number of 1s in this column * the number of 0s in this column

### Reference code

```java
class Solution {
    public int totalHammingDistance(int[] nums) {
        int len=nums.length;
        int[] bitCount = new int[32];
        if(len <= 1){
            return 0;
        }
        for(int numIndex = 0; numIndex < len; numIndex++){
            for(int bitIndex = 0; bitIndex < 32; bitIndex++){
                bitCount[bitIndex] += nums[numIndex] & 1;
                nums[numIndex] = nums[numIndex] >> 1;
                if(nums[numIndex] == 0){
                    break;
                }
            }
        }
        int oneCount = 0;
        for(int bitIndex = 0; bitIndex < 32; bitIndex++){
            oneCount += bitCount[bitIndex] * (len - bitCount[bitIndex]);
        }
        return oneCount;
    }
}
```

### Complexity analysis

Time complexity: Time complexity: O(N log ⁡C) where C is a constant, representing the maximum possible value of the number in the array.

Space complexity: O(log C)
